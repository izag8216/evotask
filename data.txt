## データ構造定義

### タスクデータ構造
```
Task {
    id: string (unique identifier)
    title: string (タスク名/種名)
    description: string (進化的特徴)
    priority: 'high' | 'medium' | 'low' (生存優先度)
    environment: 'competitive' | 'collaborative' | 'adaptive' (進化環境)
    adaptationScore: number (0-100) (適応度スコア)
    deadline: string (ISO date) (適応期限)
    status: 'active' | 'completed' | 'eliminated' (ステータス)
    generation: number (進化世代)
    mutations: array of strings (変異履歴)
    fitnessScore: number (0-100) (適応度)
    createdAt: string (ISO date)
    updatedAt: string (ISO date)
}
```

### 進化統計データ
```
EvolutionStats {
    adaptationScore: number (全体適応度)
    survivingTasks: number (生存タスク数)
    eliminatedTasks: number (淘汰タスク数)
    mutationCount: {
        strategy: number (戦略変異数)
        tactical: number (戦術変異数)
    }
}
```
